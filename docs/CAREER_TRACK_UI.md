**Career Track UI**

Эта заметка описывает, как вкладка карьеры теперь выбирает активный путь и почему игрок видит только две развилки.

**Как определяется текущий путь**

Источник правды:
- `state.playerState.currentTrackId`

Фолбэк:
- `state.game.playerState.currentTrackId`
- если ничего не найдено, используется `street`

Допущение:
- старые сейвы могут хранить несколько уже инициализированных карьерных секций одновременно
- UI не пытается показать их все
- вкладка `career` рендерит только тот путь, который помечен как текущий трек

Треки:
- `street`
- `amateur`
- `pro`

**Что видно на вкладке карьеры**

Вкладка `career` показывает только:
- текущий путь
- почему именно этот путь сейчас главный
- ближайшую цель
- компактный блок базы: зал, тренер, подготовка, контракты
- максимум две самые уместные развилки

Отдельные детальные панели не удалены:
- `careerStreet`
- `careerTeam`
- `careerPro`
- `careerTransitions`

Они остались как подробные экраны и открываются кнопками из компактной вкладки карьеры.

**Как выбираются две развилки**

Переходы по-прежнему считает `CareerTransitionEngine`.

Сначала строится полный набор transition cards:
- доступные
- заблокированные

После этого cards ранжируются, но без изменения save schema.

Для каждого перехода считаются:
- `relevanceScore`
- `timingScore`
- `eligibilityScore`

На их основе строится общий `score`.

Приоритет сортировки:
1. доступные выше заблокированных
2. выше общий `score`
3. выше `relevanceScore`
4. стабильная сортировка по `id`

На главную вкладку карьеры попадают только первые две карточки после этой сортировки.

**Что влияет на ranking**

`relevanceScore`:
- текущий трек
- тип перехода
- уличный рейтинг
- слава
- разряд
- статус в сборной
- наличие вылета из сборной
- возможность street comeback из профи

`timingScore`:
- открыт ли переход уже сейчас
- возраст рядом с окном входа или закрытия
- срочные сценарии вроде late start, вылета из сборной, pro fallback

`eligibilityScore`:
- число блокирующих причин
- доступность перехода сейчас

**Почему переход может быть скрыт**

Скрытый переход не удалён из системы.

Он просто не попал в top-2, потому что:
- не подходит текущему этапу
- имеет слишком много блокеров
- уступает другим переходам по времени
- пока не является самым вероятным шагом для игрока

Полный список всё ещё можно открыть через экран `careerTransitions`, но и там UI сначала акцентирует top-2.

**Как показываются причины**

Для transition card выводятся:
- человекочитаемое объяснение, почему шаг открыт или закрыт
- главные требования
- что будет потеряно
- что будет получено
- возможный путь возврата

Основные helper-поля presentation layer:
- `currentTrackSummary`
- `whyThisTrackIsCurrent`
- `availableTransitionReason`
- `blockedTransitionReason`

Они вычисляются поверх существующих систем и не заменяют underlying career logic.

**Совместимость**

- save schema не менялась
- `SAVE_VERSION` не поднимался
- старые transition stores не удалялись
- старые UI и runtime adapters сохранены
- если старый сейв уже содержит несколько карьерных подсистем, вкладка карьеры сама скрывает неактуальные и показывает только текущий трек
